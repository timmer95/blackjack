# User Story - Cards

## Story
As a Casino, I want to have multiple cards, so that I can let people play multiple different games at the same time.

## Acceptance criteria
A card must always have a value, name, and suits // hoeft niet te checken, compiler doet dat al
A card's value cannot change (untrue?) // hoeft niet te checken, compiler doet dat al
By default, a card is faced down.
Two similar cards are not the same object
Two different cards are not the same object
Two similar cards are similar
Two different cards are not similar
A duplication of a card is same as original object
